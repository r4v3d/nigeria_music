================================================================================
GUÍA TÉCNICA DETALLADA: CONFIGURACIONES, PASOS Y MEJORAS DE MIGRACIÓN TIDAL
================================================================================

Este documento describe con precisión técnica las configuraciones del navegador,
los scripts de sigilo, las manipulaciones de cookies, los tiempos de espera
(timeouts), las acciones ocultas y las recomendaciones específicas de qué cambiar, 
eliminar o mejorar en el script 'migrar_cuentas_tidal.py'.

--------------------------------------------------------------------------------
1. CONFIGURACIONES GENERALES DEL NAVEGADOR Y SIGILO (STEALTH)
--------------------------------------------------------------------------------

Para evadir los sistemas anti-bot de TIDAL (como DataDome y Cloudflare), el script
aplica configuraciones de bajo nivel al instanciar el navegador en Playwright:

* Argumentos de Lanzamiento de Chrome (launch_args):
  - "--disable-blink-features=AutomationControlled": Elimina la propiedad 
    `navigator.webdriver` que delata el uso de herramientas de automatización.
  - "--credentials-enable-service=false": Desactiva el servicio de almacenamiento
    de credenciales de Chrome.
  - "--password-store=basic": Configura un almacén de claves plano para evitar
    mensajes emergentes del sistema operativo sobre el guardado de claves.
  - "--disable-autofill" y "--disable-save-password-bubble": Previenen diálogos y 
    globos flotantes de Chrome que bloquean o interfieren con los clicks reales.

* Script de Sigilo Inyectado (STEALTH_SCRIPT):
  - Se ejecuta en cada frame/página antes de cargar el HTML (`add_init_script`).
  - Fuerza la eliminación de `navigator.webdriver` por JS nativo.
  - Simula propiedades de hardware normales (`navigator.hardwareConcurrency = 8`, 
    plugins falsos, lenguajes reales, y resoluciones de pantalla estándar).

* Manipulación de Cookies en Base de Datos SQLite:
  - Al copiar el perfil principal (`principal/`) para crear un perfil temporal por 
    hilo, el script abre directamente los archivos de base de datos de Chrome 
    `Default/Network/Cookies` o `Default/Cookies` usando `sqlite3`.
  - Ejecuta la consulta SQL: `DELETE FROM cookies WHERE host_key LIKE '%tidal.com%'`.
  - Motivo: Forzar a que el navegador inicie la sesión de Tidal completamente
    cerrada, evitando la reutilización de cookies de sesiones anteriores entre hilos.

2. DETALLE TÉCNICO PASO A PASO (TIMEOUTS, PASOS OCULTOS Y FLUJOS)
--------------------------------------------------------------------------------

[PASO 1] Iniciar sesión en la cuenta Tidal del cliente (Perú)
* Navegación inicial: Va a `https://account.tidal.com/profile` (Timeout: 15s, 
  wait_until="domcontentloaded").
* Espera de carga: `time.sleep(2.0)` para permitir carga de scripts pesados.
* Cierre de sesión previo (Oculto): Si detecta una sesión activa incorrecta,
  ejecuta JS nativo para posicionar el cursor sobre "Mi cuenta" y pulsa 
  "Cerrar sesión". Espera 2.0s a que redireccione.
* Login: Rellena usuario y contraseña usando velocidad humana. Si el login pide 
  un código en vez de contraseña (debido a sospechas de bot), busca el botón 
  "Inicia sesión con contraseña" para forzar el flujo clásico.

[PASO 2] Cambiar correo a cakeseller1234 con puntos aleatorios
* Objetivo: Asignar un correo temporal para liberar el correo real del cliente.
* Acción: Genera una combinación aleatoria de puntos en "cakeseller1234@gmail.com".
* Relleno: Introduce el nuevo correo en la configuración del perfil del cliente.
* Verificación IMAP: Espera que llegue el código al buzón. 
  - Frecuencia de chequeo: Consulta IMAP cada 12 segundos.
  - Reintentos: Máximo 8 reintentos (1.5 minutos de espera total).
  - Filtro estricto: Valida que la combinación exacta de puntos coincida en las 
    cabeceras `To`, `Delivered-To` o `Envelope-To`.

[PASOS 3 & 4] Exportando playlists a CSV vía TuneMyMusic
* Navegación: Abre `https://www.tunemymusic.com/es/transfer`.
* Carga de archivos (Oculto):
  - Limpia el DOM eliminando banners molestos con JS.
  - Sube automáticamente las cookies del perfil.
  - Al pulsar en la exportación, Playwright maneja el evento de descarga 
    (`page.expect_download()`) y guarda el archivo CSV en la carpeta 
    `descargas/` con el nombre formateado del cliente.
* Timeout crítico: Espera de carga de la lista de reproducción (hasta 30 segundos).

[PASO 5] Eliminando la cuenta Tidal del cliente
* Navegación: `https://login.tidal.com/delete-account` (o ruta equivalente).
* Confirmaciones (Oculto): Pasa a través del asistente de 4 pasos de confirmación.
  - Paso 1: Activa el checkbox "He leído lo anterior" vía JS nativo.
  - Paso 2 a 4: Clica en "Continuar" usando evaluates nativos si los elementos 
    están dentro de Shadow DOMs u overlays.
* IMAP Código de Eliminación:
  - Conecta a `cakeseller1234@gmail.com` via IMAP.
  - Espera un código de 5 dígitos (ej: 28728).
  - Timeout: Máximo 8 reintentos con 12 segundos de pausa entre ellos (1.5 min total).
  - Relleno: Detecta las 5 cajas de texto individuales de Tidal y simula pulsaciones
    de teclado numérico para burlar detectores de pegado.

[PASO 6] Creando nueva cuenta Tidal del cliente (Nigeria)
* Conexión: Se configura el proxy residencial de Nigeria.
* Reputación previa (Acción Oculta): Abre primero `tidal.com/pricing` y espera 3.5s
  para simular navegación de un usuario interesado antes de ir a registrarse.
* Registro: Registra el correo real del cliente con contraseña temporal.
* Verificación IMAP: 
  - Busca el correo de bienvenida/registro de Tidal.
  - Frecuencia: Cada 12 segundos, máximo 5 intentos (1 minuto).
  - Rellena el código de 6 dígitos simulando teclado.

[PASO 7] Transfiriendo música del CSV a la nueva cuenta Tidal
* Navegación: Abre TMM en Perú (PE).
* Login en TMM (Paso Oculto):
  - Clica en "Subir archivo" y carga el CSV temporal.
  - Clica en "Elige destino" -> "Tidal".
  - Si aparece el popup de autenticación de Tidal, Playwright captura la ventana
    emergente (`page.context.expect_page()`), clica en "Sí, continuar" y luego 
    en "Continuar" (con hasta 3 reintentos).
  - Cierra de forma forzada con JS nativo el modal "Name your TIDAL Accounts" 
    enviando el evento `Escape`.
* Transferencia: Clica en "Comenzar a transferir música" (Timeout de espera: 20s).

[PASO 8] Solicitando restablecimiento de contraseña para la nueva cuenta
* Navegación: `https://login.tidal.com/resetpass`.
* Envío: Introduce el correo real del cliente y presiona enviar.
* Bloqueo serializado: Utiliza `reset_lock` para evitar que varios hilos envíen
  solicitudes de restablecimiento al mismo tiempo, previniendo el rate limiting de Tidal.

[PASO 9] Creando invitación familiar en la cuenta del titular
* Navegación: Inicia sesión con la cuenta titular en `https://account.tidal.com/family`.
* Envío de invitación (Oculto): Detecta cuántos espacios libres quedan (máx 5).
  - Clica en "Invitar".
  - Rellena el correo real del cliente y envía.
* Sincronización: El primer hilo que toma el control del navegador familiar
  agrega TODOS los correos del lote actual a la vez para ahorrar tiempo y recursos,
  y los demás hilos simplemente saltan este paso marcándolo como "ya invitado".

[PASO 10] Buscando enlace de reinicio de contraseña y cambiando contraseña
* IMAP Enlace de Reset:
  - Busca el correo de reset de Tidal.
  - Frecuencia: Cada 15 segundos, con un máximo de 8 intentos.
  - Extrae el enlace del cuerpo del correo (ej: `https://login.tidal.com/resetpass/...`).
  - **Filtro estricto (NUEVO)**: Compara que el correo destinatario coincida 
    exactamente con los puntos (ej. `getmush.room3819` no colisiona con `getmus.hroom381.9`).
* Cambio de contraseña: Abre el enlace e introduce `nueva_contrasena`.

[PASO 11] Aceptando invitación familiar en el correo del cliente
* IMAP Enlace de Invitación:
  - Conecta a la cuenta de Gmail del cliente.
  - Busca el correo de invitación con el enlace "Join Family".
  - Frecuencia: Cada 12 segundos, máximo 5 intentos (1 minuto).
  - Abre el enlace, rellena la nueva contraseña configurada en el Paso 10
    e inicia sesión para aceptar la vinculación al grupo familiar.

3. QUÉ MANTENER, ELIMINAR, CAMBIAR O MEJORAR
--------------------------------------------------------------------------------

A nivel de código, estas son las instrucciones concretas para optimizar el script:

### A MANTENER (No tocar):
1. SQLite cookie cleaner: La limpieza de la base de datos de cookies de Chrome 
   a nivel de archivo antes del inicio es sumamente eficiente y evita que las 
   sesiones de Tidal se mezclen entre perfiles clonados.
2. Inyección de STEALTH_SCRIPT: Es obligatorio mantenerlo para pasar el control de
   bot de Playwright.
3. SystemLock en TuneMyMusic: Evita baneos por concurrencia masiva en la misma IP/TMM.

### A ELIMINAR (Código obsoleto o innecesario):
1. `optimizar_pagina` (Línea 1478): 
   - Actualmente tiene un `return` directo y está desactivada. 
   - Razón: Omitir la carga de imágenes/fuentes para ahorrar datos rompía los 
     flujos visuales de Tidal y TuneMyMusic, provocando que los botones no fueran
     clicables. Debe eliminarse por completo del script para limpiar el código.
2. Comprobación manual de cookies en TMM (Líneas redundantes):
   - Eliminar los bloques redundantes de código que intentan forzar la inyección 
     manual de cookies en TMM. Playwright ya gestiona esto correctamente con el 
     perfil temporal copiado.

### A CAMBIAR (Modificar lógica existente):
1. Activar el aborto de barreras en fallos:
   - Modificar la función `registrar_error_y_abortar` para invocar 
     `self.abort_all_barriers()`.
   - Código actual:
     ```python
     def registrar_error_y_abortar(self, exception, paso_nombre):
         if not self.cuenta_abortada:
             print(f"  [ERROR] {paso_nombre} falló para {self.client_email}: {exception}")
             self.cuenta_abortada = True
             self.abort_reason = f"Error en {paso_nombre}: {exception}"
             ...
     ```
   - Código propuesto:
     ```python
     def registrar_error_y_abortar(self, exception, paso_nombre):
         if not self.cuenta_abortada:
             print(f"  [ERROR] {paso_nombre} falló para {self.client_email}: {exception}")
             self.cuenta_abortada = True
             self.abort_reason = f"Error en {paso_nombre}: {exception}"
             self.abort_all_barriers() # <-- CAMBIO CRÍTICO: Rompe las barreras para que otros hilos no se queden colgados.
             ...
     ```
2. Remplazar la lógica de re-solicitud de restablecimiento de contraseña:
   - En el Paso 10, si el enlace no llega, el script intenta volver a solicitar 
     el restablecimiento en un bucle que puede causar rate limit. Reducir los 
     intentos de re-solicitud de 3 a 1.

### A MEJORAR (Nuevas funcionalidades sugeridas):
1. Implementación de una Máquina de Estado Persistente (Resumabilidad):
   - Crear un archivo `progreso_migraciones.json` en la raíz del proyecto.
   - Cada vez que un hilo complete con éxito un paso (ej. Paso 5 completado), 
     escribe en el JSON: `{"cliente@gmail.com": 5}`.
   - Al iniciar, el script debe leer este archivo y asignar dinámicamente el 
     `start_step` de cada cuenta individualmente, en lugar de forzar a que todo 
     el lote inicie desde el mismo paso con `--start-step`.
2. Manejador de Salida Limpia (Signal Handler):
   - Importar `signal` y registrar `signal.SIGINT` (Ctrl+C).
   - Al detectar la interrupción, ejecutar una función que barra la carpeta 
     `C:\Users\prett\.tidal_migrator\perfiles` buscando directorios que comiencen 
     con `temp_profile_*` y eliminarlos, evitando dejar gigabytes de basura.
3. Alertas por webhook (Telegram/Discord):
   - Al ser un proceso largo (casi 50 minutos para 5 cuentas), añadir una función
     opcional para enviar una notificación cuando una cuenta falle o complete
     exitosamente todo el proceso.
